# ETL и анализ e-commerce данных (Hadoop, Spark, Hive)

## Описание
Реализован ETL-пайплайн для обработки e-commerce данных с использованием Hadoop-экосистемы.

## Бизнес-цель
Подготовить данные для анализа продаж и поведения пользователей, выявить ключевые метрики и проблемы качества данных.

## Стек
- HDFS
- Hive
- Spark (SQL, DataFrame API)
- Parquet

## Архитектура
RAW → Hive → Spark → Data Mart

## Что сделано
- загружены данные в HDFS
- созданы внешние таблицы в Hive
- реализован ETL-процесс обработки данных
- выполнена дедупликация данных
- рассчитаны производные метрики (total_price)
- построена Snowflake-модель данных

## Анализ данных
- анализ распределения цен по категориям
- анализ качества данных (NULL, пропуски)
- выявление аномалий

## Выводы
- обнаружены пропуски в ключевых полях (~X%)
- выявлены категории с наибольшей средней ценой
- найдены аномальные значения в стоимости товаров

## Результат
Подготовлен набор данных, пригодный для BI-аналитики и построения отчетности.
